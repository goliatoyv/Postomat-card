# 📦 Аналітика Поштоматів — Geo-Analytics Suite (Нова Пошта)

Інтерактивний дашборд аналітики мережі поштоматів у стилі Tableau.
Перероблено з автономного HTML-прототипу в повноцінний **React + TypeScript + Vite** проєкт із компонентною архітектурою.

## 🚀 Запуск

```bash
npm install
npm run dev        # режим розробки → http://localhost:5173
npm run build      # типчек + продакшн-збірка у dist/
npm run preview    # локальний перегляд зібраного бандлу
npm run lint       # перевірка типів (tsc --noEmit)
```

## 🧱 Структура проєкту

```
index.html                  Точка входу Vite
src/
  main.tsx                  Монтування React-застосунку
  index.css                 Tailwind + кастомні стилі (карта, лассо, анімації)
  types.ts                  Типи доменної моделі (Cell, PostomatDetail, Filters…)
  lib/format.ts             Хелпери: ГПВЧ, форматування, колірна шкала, point-in-polygon
  data/mockData.ts          Генерація mock-даних (гео-сітка + профілі поштоматів)
  App.tsx                   Кореневий компонент, стан фільтрів і вибору
  components/
    TopBar.tsx              Глобальні фільтри (гео-ієрархія, сітка, інфра-перемикачі)
    MapPanel.tsx            SVG-карта з гео-сіткою, маркерами та лассо
    DetailPanel.tsx         Картка поштомата — 7 блоків аналітики
    Heatmap.tsx             Теплова карта активності 7×24
```

## ✨ Функціонал

- **Верхня панель:** гео-ієрархія (Область→Район→Місто→Тип НП), розмір сітки
  `100м/300м/500м/1км` (реально перебудовує сітку), діапазон клієнтів,
  4 незалежні інфра-перемикачі `Всі/Є/Немає` (Поштомат, Вантажне, Відділення, ПУДО).
- **Карта (60%):** квадратна гео-сітка з градієнтом щільності (зелений→червоний),
  AND-фільтрація, маркери інфраструктури, виділення зони (лассо).
- **Картка поштомата (40%):** 7 блоків — динаміка з перемикачем доходу
  (валовий/локальний), концентрація + ТОП-20, профіль відправлень, операційна
  ефективність, сегментація (RFM/гео-тип), спец-вантажі та фінанси, heatmap 7×24.

## 🗃️ Модель даних

Mock-дані змодельовано за структурою джерел Tableau:
`Report_HeatMap_refact_WhData_v`, `…_Rentability_and_Rent_v`,
`…_dim_bielik_dev_v`, `Report_HeatMap_UserPoints_not_in_Buffer_v`.
Детерміновані сіди гарантують стабільність даних між перемиканнями фільтрів.

## 📁 `standalone.html`

Початковий однофайловий прототип (без збірки) збережено як `standalone.html` —
його можна відкрити подвійним кліком у браузері без жодних залежностей.
